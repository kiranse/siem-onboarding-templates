Contributing Guidelines

Thank you for your interest in contributing to this repository.

Purpose

This repository provides generic templates and examples for onboarding log sources into SIEM platforms. Contributions should aim to improve clarity, usability, and consistency.

How to Contribute

You can contribute in the following ways:

Improve existing templates
Add new reusable templates
Fix formatting or documentation issues
Suggest enhancements or best practices
Contribution Guidelines
Keep all content generic and sanitized
Do not include:
internal company information
production endpoints
credentials or secrets
environment-specific configurations
Follow a clear and consistent structure
Use simple and readable formatting
Submitting Changes
Fork the repository
Create a new branch
Make your changes
Submit a pull request with a clear description
Notes

All contributions should align with the goal of making SIEM onboarding more structured and easier to follow across different environments.

Thank you for contributing.
